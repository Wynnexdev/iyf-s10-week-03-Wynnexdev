## Project Description
This project demonstrates the use of developer tools such as the command line interface (CLI) and Git version control. The objective of the assignment was to practice navigating the file system using terminal commands, creating and managing files and directories, and maintaining a structured Git workflow with commits, branching, and merging.

The project repository also documents the commands used during the process and includes a professional project structure.

## Lessons Learned
During this project I learned how to:

- Navigate the file system using terminal commands
- Create and manage directories and files using CLI tools
- Search for files and content using commands such as `find` and `grep`
- Initialize and manage a Git repository
- Use meaningful commit messages to track project progress
- Create branches for new features and merge them into the main branch
- Push local changes to a remote repository on GitHub

## Challenges Faced
One challenge I encountered was understanding how Git branching and merging works. Initially, switching between branches and merging changes created confusion, especially when managing commit history.

To solve this, I practiced the workflow multiple times and reviewed the Git command documentation to better understand how branches operate and how merges combine changes.

Another challenge was remembering various terminal commands. I addressed this by documenting the commands used during the project in a `terminal-log.md` file.
